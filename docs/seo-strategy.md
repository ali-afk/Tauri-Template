## Meta Component

The `Meta` component provides type-safe SEO metadata for every page.

**Location:** `src/lib/components/shared/Meta.svelte`

### Usage

**For regular (indexable) pages:**
```svelte
<Meta
  title="Councils"
  description="Step into the arena of international diplomacy..."
  pageURI="/councils"
  keywords={["Councils", "Diplomatic Simulation"]}
/>
```

**For non-indexed pages (like coming-soon):**
```svelte
<Meta
  title="Coming Soon"
  description="NoIndex"
  pageURI="/"
/>
```

### What It Generates

The Meta component automatically creates:

1. **Page title:** `{title} | BritMUN XI`
2. **Meta description:** For search results preview
3. **Keywords:** Global keywords + page-specific keywords
4. **Canonical URL:** Points to the official page location
5. **Open Graph tags:** For social media sharing (Facebook, LinkedIn)
6. **Twitter Card tags:** For Twitter/X sharing
7. **Robots tag:** `noindex` for non-indexed pages

### Global Keywords

Defined in `Meta.svelte`:
- BritMUN XI
- Model United Nations Bahrain
- BSB MUN 2026
- British School of Bahrain

**Every page** automatically includes these global keywords.

### Adding Page-Specific Keywords

```svelte
<Meta
  title="Councils"
  description="..."
  pageURI="/councils"
  keywords={["UN Councils", "UNSC", "UNICEF", "Debate"]}
/>
```

**Best practices:**
- 3-5 keywords per page
- Focus on what makes THIS page unique
- Use terms students/parents might search for
- Avoid keyword stuffing

## JSON-LD Structured Data

Structured data helps search engines understand your site and show rich results.

**Location:** `src/lib/data/shared/meta.ts`

### Organization Schema

Tells search engines about BritMUN as an organization.

```json
{
  "@type": "Organization",
  "name": "BritMUN XI",
  "description": "Model United Nations conference at BSB",
  "logo": "https://britmun.netlify.app/icon-512.png",
  "email": "britmun@thebsbh.com",
  "sameAs": [
    "https://www.instagram.com/britmun.bh/",
    "https://www.tiktok.com/@britmun"
  ]
}
```

### Event Schema

Tells search engines about the conference as an event.

```json
{
  "@type": "EducationalEvent",
  "name": "BritMUN XI",
  "description": "Model United Nations Conference 2026",
  "startDate": "2026-01-30",
  "endDate": "2026-01-31",
  "eventAttendanceMode": "OfflineEventAttendanceMode",
  "eventStatus": "EventScheduled",
  "location": {
    "@type": "Place",
    "name": "British School of Bahrain",
    "address": {
      "@type": "PostalAddress",
      "addressLocality": "Hamala",
      "addressCountry": "BH"
    }
  },
  "organizer": {
    "@type": "Organization",
    "name": "British School of Bahrain MUN Team"
  },
  "offers": {
    "@type": "Offer",
    "price": "25.000",
    "priceCurrency": "BHD"
  }
}
```

### Benefits of Structured Data

- **Rich results:** Enhanced search listings with event details
- **Knowledge panels:** Can appear in Google's knowledge graph
- **Event discovery:** Shows up in Google Events, event aggregators
- **Social sharing:** Better previews when shared on social media

### Testing Structured Data

Use [Google Rich Results Test](https://search.google.com/test/rich-results):
1. Enter your site URL
2. Check for errors/warnings
3. Preview how it appears in search results

## Open Graph & Twitter Cards

Controls how your site appears when shared on social media.

**Generated by:** `Meta.svelte`

### What's Included

**Open Graph (Facebook, LinkedIn, WhatsApp):**
```html
<meta property="og:title" content="Councils | BritMUN XI">
<meta property="og:description" content="Step into the arena...">
<meta property="og:url" content="https://britmun.netlify.app/councils">
<meta property="og:type" content="website">
<meta property="og:image" content="https://britmun.netlify.app/icon-512.png">
```

**Twitter Card (Twitter/X):**
```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Councils | BritMUN XI">
<meta name="twitter:description" content="Step into the arena...">
<meta name="twitter:image" content="https://britmun.netlify.app/icon-512.png">
```

### Social Image

Currently uses `icon-512.png` (512×512).

**For better social sharing:**
1. Create a 1200×630 image with:
   - BritMUN logo
   - Conference name and year
   - Dates or tagline
2. Save as `static/og-image.png`
3. Update `Meta.svelte` to use this image

**Image requirements:**
- Recommended: 1200×630 px (1.91:1 ratio)
- Min: 600×315 px
- Max: 8MB
- Format: PNG or JPG

### Testing Social Cards

**Facebook Debugger:** https://developers.facebook.com/tools/debug/
**Twitter Card Validator:** https://cards-dev.twitter.com/validator
**LinkedIn Post Inspector:** https://www.linkedin.com/post-inspector/

## SEO Checklist

Use this checklist when creating new pages or updating SEO.

### Page-Level SEO

- [ ] **Title tag:** Unique, descriptive, under 60 characters
- [ ] **Meta description:** Compelling, 150-160 characters
- [ ] **Keywords:** 3-5 relevant keywords (global + local)
- [ ] **Canonical URL:** Correct page path
- [ ] **Heading hierarchy:** Single H1, proper H2-H6 structure
- [ ] **Image alt text:** Descriptive alternative text for all images

### Technical SEO

- [ ] **Sitemap:** Page included in `/sitemap.xml`
- [ ] **robots.txt:** Not blocking the page
- [ ] **Mobile-friendly:** Responsive design works on phones
- [ ] **Page speed:** Fast load times (test with PageSpeed Insights)
- [ ] **HTTPS:** Site served over secure connection
- [ ] **Structured data:** Valid JSON-LD (test with Rich Results Test)

### Content SEO

- [ ] **Unique content:** Not duplicated from other pages/sites
- [ ] **Quality writing:** Clear, helpful, well-organized
- [ ] **Internal links:** Links to other relevant pages on site
- [ ] **External links:** Links to official resources (marked with `rel="noopener noreferrer"`)
- [ ] **Accessibility:** Semantic HTML, ARIA labels, skip links

### Social Media

- [ ] **Open Graph tags:** Present and correct
- [ ] **Twitter Card tags:** Present and correct
- [ ] **Social image:** Attractive share image (1200×630)
- [ ] **Test previews:** Validated with Facebook/Twitter tools

## Common SEO Issues & Fixes

### Issue: Page not showing in Google

**Possible causes:**
- Not yet indexed (wait 1-2 weeks)
- Blocked by robots.txt
- NoIndex meta tag present

**Fix:**
1. Check `robots.txt` → should allow the page
2. Check Meta component → should NOT use `description="NoIndex"`
3. Submit sitemap to Google Search Console
4. Request indexing via GSC

---

### Issue: Poor search ranking

**Possible causes:**
- Weak content
- Missing keywords
- No backlinks
- Slow page speed
- Not mobile-friendly

**Fix:**
1. Improve content quality and length
2. Add relevant keywords naturally
3. Get links from school website, social media
4. Optimize images, enable caching
5. Test and fix mobile responsiveness

---

### Issue: Structured data errors

**Possible causes:**
- Missing required fields
- Invalid date format
- Incorrect schema type

**Fix:**
1. Test with [Rich Results Test](https://search.google.com/test/rich-results)
2. Check `meta.ts` for required fields
3. Validate date formats (YYYY-MM-DD)
4. Ensure `@type` is correct

## Updating SEO for New Conference

When creating BritMUN XII (next year):

1. **Update SiteProperties:**
   - `year: "2027"`
   - `britmunYear: { roman: "XII", decimal: "12" }`
   - `eventDate: { start: "2027-01-29", end: "2027-01-30" }`

2. **Update Meta global keywords:**
   - Change "BSB MUN 2026" → "BSB MUN 2027"
   - Change "BritMUN XI" → "BritMUN XII"

3. **Structured data auto-updates** via SiteProperties

4. **Test everything:**
   - Rich Results Test
   - Social media previews
   - Sitemap validation
